1단계:
단순 표시 로직 (초기 구현)

처음에는 A* 알고리즘이 반환한 경로 좌표 리스트를 받아,
각 좌표를 월드 좌표로 변환한 뒤 하이라이트 프리팹을 Instantiate하여 표시하는 단순한 구조로 구현
경로를 다시 표시할 때는 기존에 생성된 오브젝트를 Destroy하여 초기화

- List로 생성된 오브젝트 관리
- Instantiate → Destroy 반복
- 기능 검증 목적의 가장 단순한 구조

2단계:
책임 분리 (계산 vs 표현)

이후 경로 계산 로직(A*)과 시각적 표현 로직이 섞이는 문제를 방지하기 위해,
경로 계산은 별도의 시스템에서 수행하고 하이라이터는 좌표 리스트를 입력받아 표시만 담당하도록 역할을 분리

- 하이라이터는 규칙을 모른다
- 입력(List<Vector2Int>)만 받아서 출력(시각화)만 수행
- 구조적 설계 개선

3단계:
성능 개선을 위한 오브젝트 풀 도입

경로 미리보기와 같은 반복 호출 상황에서
Instantiate / Destroy의 비용이 누적되는 문제가 있음.
생성과 삭제 대신 오브젝트 풀링 방식으로 구조를 개선
하이라이트 오브젝트는 한 번 생성한 뒤 SetActive를 통해 재사용하도록 변경

- Destroy 제거
- activeCount 개념 도입
- 재사용 기반 구조

4단계:
레이어 개념 도입 (표현 확장 대비)

단순 경로 표시 외에도 이동 가능 범위, 이동 불가 영역, 힌트 표시 등
다양한 시각적 표현이 필요해질 것을 고려하여 하이라이트를 레이어 개념으로 분리

- 구조의 확장이 가능하도록 설계한다는 것이 포인트

5단계: 딕셔너리 기반 레이어별 풀 관리

레이어가 증가함에 따라 레이어별로 오브젝트 풀이 섞이지 않도록
Dictionary<Layer, List<GameObject>> 구조를 사용해 레이어 단위로 풀과 사용 개수를 분리 관리
이를 통해 특정 레이어만 갱신하거나 초기화

ShowCells 함수는 보드와 좌표 리스트, 레이어를 입력받아
해당 레이어의 기존 표시를 초기화한 뒤,
좌표 유효성 검사 → 월드 좌표 변환 → 레이어 전용 오브젝트 풀에서 하이라이트를 활성화하여 배치
이 과정에서 생성 비용은 최소화되고, 레이어 간 표시 로직은 서로 독립적으로 동작한다.
