# Relic Jackal

개인 게임 프로젝트\
A* 기반 의사결정 구조 연습용 프로젝트

---

## Project Overview

**Relic Jackal**은\
A* 알고리즘을 실제 게임 구조 안에서 사용해 보기 위해 진행 중인\
**개인 학습용 프로젝트**입니다.

턴 기반 구조 안에서\
플레이어의 이동과 선택이 어떤 결과를 만들어내는지 확인하는 것을 목표로 하고 있습니다.

현재는 전체 게임 완성이 아닌,\
**핵심 구조만 확인할 수 있는 버티컬 슬라이스 단계**를 제작 중입니다.

---

## Project Goal

- A* 알고리즘을 게임 이동 로직에 적용
- 선택에 따라 결과가 달라지는 구조 설계 연습
- 전투와 판단 흐름을 단계적으로 나누어 구현해 보기

---

## Design Notes

### 1. A* 기반 이동 구조

플레이어의 이동은\
그리드 기반 맵 위에서 A* 알고리즘으로 경로를 계산합니다.

- 이동 가능 경로 계산
- 위험 지역을 고려한 경로 선택
- 이동 결과가 이후 전투에 영향을 주는 구조

A* 알고리즘을\
단순한 길찾기가 아닌 **플레이 선택의 근거**로 사용해 보는 것이 목적입니다.

---

### 2. Battle Phase 분리

전투는 한 번에 처리하지 않고,

- 이동 / 탐색
- 전투
- 결과 정리

와 같이 단계를 나누어 구성하고 있습니다.

턴제 구조와 전투 단계를 분리해\
플레이가 단조롭지 않도록 하는 방향을 실험 중입니다.

---

### 3. 선택과 결과 구조

플레이어의 선택에는 항상 결과가 따르도록 설계했습니다.

- 위험한 선택 → 손해 가능성 증가
- 안전한 선택 → 진행은 느리지만 안정적

정확한 밸런스보다는\
**선택 → 결과 흐름을 직접 만들어보는 것**에 초점을 두고 있습니다.

---

### 4. 실패를 허용하는 흐름

이 프로젝트에서는\
실패를 바로 게임 오버로 처리하지 않습니다.

- 후퇴
- 자원 손실
- 목표 미달 상태로 생존

등의 결과가 발생하도록 설계하여,\
실패도 하나의 플레이 결과로 경험하도록 구성했습니다.

---

## Why This Project

Relic Jackal은\
완성된 게임을 목표로 한 프로젝트가 아니라,

- A* 알고리즘을 실제 게임 구조에 적용해 보고
- 기획자가 시스템을 어떻게 설계해야 하는지 연습하기 위한 프로젝트입니다.

기능 구현보다\
**구조를 이해하고 설계하는 경험**을 쌓는 데 목적이 있습니다.
